# 2YOU Standard Workflows and Prompts

## Purpose

This file contains ready-to-use workflow patterns and activation prompts for the 2YOU system.

These are not theoretical examples.
They are intended to be copied, adapted, and used as the standard method for launching common 2YOU jobs.

---

## Workflow 1: Offer Clarification Sprint

### When to use
- offer is too broad
- audience fit is unclear
- pricing logic feels weak
- sales narrative lacks sharpness

### Activation prompt

```markdown
Activate the repository under the 2YOU OS for an Offer Clarification Sprint.

Objective: [business objective]
Current offer: [offer]
ICP: [ICP]
CTA: [CTA]
KPI: [KPI]
Mode: [Caixa / Validação / Escala]
Constraints: [constraints]

Required sequence:
1. 2YOU Chief Orchestrator classifies the request
2. 2YOU Context Keeper retrieves current initiative truth
3. apply the 2YOU decision gates
4. activate Studio Producer, Proposal Strategist, and Brand Guardian as needed
5. use Senior Project Manager to convert the chosen path into a tactical next step

Deliverable required:
- clarified offer
- selected angle
- rejected alternatives
- CTA logic
- KPI logic
- next handoff
```

### Expected output
- clearer offer structure
- sharper buyer fit
- commercial narrative fit
- next tactical step

- duplication is everywhere
- handoffs are broken
- the team is busy and unclear at the same time

### Activation prompt

```markdown
Activate the repository under the 2YOU OS for a Project Recovery Sprint.

Objective: recover alignment and restore execution coherence
Current initiative: [initiative]
Known issues: [issues]
Mode: [usually Caixa or Validação]
Constraints: [constraints]

Required sequence:
1. Chief Orchestrator classifies the request
2. Context Keeper retrieves current initiative truth
3. apply scope reality and operational activation gates
4. activate Project Shepherd, Workflow Optimizer, and Agents Orchestrator
5. activate Senior Project Manager to rebuild realistic task structure if necessary
6. output corrected owner map, priority order, blocked items, descoped items, and next handoff
```

### Expected output
- reduced chaos
- reset priorities
- repaired workflow structure

## Universal Mini Prompt

Use this when you need a fast but disciplined activation.

```markdown
Operate under the 2YOU OS.
Keep all agents.
Chief Orchestrator first.
Context Keeper second.
Apply gates.
Choose one mode.
Lock objective, offer, ICP, CTA, KPI, owner, and next handoff.
Then activate only the needed workflow.
```

---

## Universal Red Flag Prompt

Use this when the system starts drifting into noise.

```markdown
Pause execution and diagnose under the 2YOU OS.
Identify whether the current problem is strategic, tactical, operational, or orchestration-based.
Query Context Keeper.
Apply decision gates.
List contradictions, duplicated work, unclear ownership, CTA/KPI mismatches, and unnecessary scope.
Return a recovery plan with descopes, priorities, and next handoff.
```

## Final Note

These workflows should become the default activation patterns for the repository.

Otherwise the system will slowly drift back toward ad-hoc brilliance, which is just a poetic way to describe preventable disorder.
