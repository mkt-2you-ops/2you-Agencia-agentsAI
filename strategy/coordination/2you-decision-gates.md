# 2YOU Decision Gates

## Purpose

These gates sit above the existing NEXUS quality gates. NEXUS validates workflow and execution quality. 2YOU validates whether the work should happen, how far it should go, and under what operating logic.

The sequence is deliberate.

A request may be beautifully executed and still fail the business logic.

---

## Gate 0: Business Readiness Gate

### Question
Do we have enough commercial clarity to activate meaningful work?

### Required fields
- Objective
- Offer
- ICP
- Promise
- CTA
- KPI
- Owner
- Time horizon

### Pass criteria
- At least 7 of 8 fields defined
- No contradiction between CTA and KPI
- No contradiction between offer and ICP

### Fail actions
- reframe request
- reduce scope
- send to strategy layer first

### Gate keeper
- 2YOU Chief Orchestrator
- 2YOU Context Keeper

---

## Gate 1: Cash / Validation / Scale Gate

### Question
Which mode is this work actually in?

### Available modes
- Caixa
- Validação
- Escala

### Pass criteria
A single mode is chosen and justified.

### Fail actions
- do not start mixed logic execution
- split initiative into separate tracks if needed

### Gate keeper
- 2YOU Chief Orchestrator

---

## Gate 2: Scope Reality Gate

### Question
Is the requested scope compatible with timeline, capacity, budget logic, and available assets?

### Pass criteria
- scope is small enough to be owned
- dependencies are visible
- asset requirements are known
- timeline is not fantasy

### Fail actions
- descope
- reduce channels
- reduce deliverables
- shorten the cycle
- switch mode if needed

### Gate keeper
- Project Shepherd
- Senior Project Manager
- 2YOU Chief Orchestrator

---

## Gate 3: Tactical Architecture Gate

### Question
Do we know how this strategy becomes execution?

### Required elements
- funnel or journey stage
- channel sequence
- asset list
- owner per stage
- KPI logic
- definition of done

### Pass criteria
- there is a tactical route, not just an idea
- every major deliverable has destination and owner

### Fail actions
- send to tactical planning
- block execution specialists until route is clear

### Gate keeper
- Senior Project Manager
- Sprint Prioritizer
- Studio Producer

---

## Gate 4: Operational Activation Gate

### Question
Is the work briefed well enough for specialists to execute without guesswork?

### Required elements
- objective
- CTA
- ICP
- asset purpose
- owner
- deadline
- handoff target

### Pass criteria
- specialists receive a complete brief
- 2YOU Context Keeper confirms continuity

### Fail actions
- reject activation
- return to tactical architecture
- rewrite brief using the 2YOU handoff template

### Gate keeper
- 2YOU Context Keeper
- functional lead agent

---

## Gate 5: Quality & Reality Gate

### Question
Did the output solve the job, not just complete the task?

### Pass criteria
- matches objective
- matches CTA and stage
- technically sound
- brand-consistent enough for mode
- measurable or testable

### Fail actions
- revise
- reduce
- reposition
- hold from scaling

### Gate keeper
- Reality Checker / EvidenceQA / Test Results Analyzer
- Analytics Reporter when performance is relevant

---

## Gate 6: Performance Learning Gate

### Question
What did we learn, and how does that change the next cycle?

### Required outputs
- KPI result
- what worked
- what failed
- what changes next
- what becomes standard
- what gets deprecated

### Pass criteria
- learning is recorded in reusable form
- Context Keeper updated
- next cycle has a more informed starting point

### Fail actions
- do not scale
- do not archive as success
- request post-mortem or synthesis

### Gate keeper
- Analytics Reporter
- Feedback Synthesizer
- 2YOU Context Keeper

---

## Decision Ladder

Use this order whenever a gate fails.

1. reframe
2. descope
3. split
4. pause
5. escalate
6. kill

Do not jump straight to “build more.” That is usually the dumbest option dressed as momentum.

## Fast Interpretation Guide

- **PASS**: proceed
- **CONDITIONAL PASS**: proceed with explicit constraints
- **FAIL**: do not proceed as requested

Conditional pass should be common.
That is what adult operations look like.
