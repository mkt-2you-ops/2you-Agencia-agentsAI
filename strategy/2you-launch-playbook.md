# 2YOU Launch Playbook

## Purpose

This playbook is the standard operating guide for launching and running the repository under the 2YOU system.

It turns the current repository from a set of powerful agents plus governance files into a **repeatable activation model**.

Use this playbook whenever you want the system to behave like an operator instead of a prompt collection.

---

## What This Playbook Assumes

The following files already exist in the repository:
- `strategy/2you-operating-system.md`
- `specialized/2you-chief-orchestrator.md`
- `specialized/2you-context-keeper.md`
- `strategy/coordination/2you-decision-gates.md`
- `strategy/coordination/2you-handoff-template.md`
- `strategy/coordination/2you-agent-routing-matrix.md`
- the 2YOU bridge files for core and high-output agents

If those are missing, install them first.

## The 2YOU Activation Principle

Every serious workflow begins in this order:

1. **2YOU Chief Orchestrator** classifies the request
2. **2YOU Context Keeper** retrieves current initiative truth
3. **Decision Gates** validate readiness
4. **Strategy / Tactic / Operation / Orchestration** are separated
5. Only then are downstream agents activated
6. Learnings are pushed back into context memory

Do not reverse this.

Starting with execution feels fast and usually creates expensive rework.

---

## The Three Operating Modes

### Caixa
Use when the system must prioritize revenue speed, directness, and simplicity.

Best for:
- offer monetization
- quick campaign launches
- sales asset creation
- lean funnel deployment

### Validação
Use when the system must reduce risk through testing and learning.

Best for:
- angle testing
- offer validation
- new ICP exploration
- experimental content and funnel hypotheses

### Escala
Use when the system already has signal and now needs repeatability.

Best for:
- structured content systems
- channel expansion with discipline
- reporting systems
- automation and throughput improvements

Choose one mode per initiative.

## The Five Required Inputs

Before launching any serious workflow, confirm at minimum:
- objective
- offer
- ICP
- CTA
- KPI

If you cannot answer those clearly, do not launch downstream execution.

For stronger workflows, also lock:
- promise
- owner
- deadline
- constraints
- operating mode

## Standard Launch Sequence

### Step 1: Bootstrap the System
Use `strategy/2you-system-bootstrap.md`.

### Step 2: Classify the Request
2YOU Chief Orchestrator determines:
- primary layer
- business stage
- operating mode
- whether the request should proceed, reframe, descope, pause, or escalate

### Step 3: Retrieve Current Truth
2YOU Context Keeper returns:
- objective
- offer
- ICP
- CTA
- KPI
- active assets
- constraints
- live learnings

### Step 4: Apply Decision Gates
Use the 2YOU gate system before activating specialists.

### Step 5: Route to the Correct Workflow
Use one of the workflow types below:
- Offer
- Campaign
- Proposal
- Funnel
- Content Operation
- Project Recovery

### Step 6: Execute Through Handoffs
Use the 2YOU handoff template for each major transition.

### Step 7: Validate and Learn
Use QA, analytics, and context update as the closing loop.

## Common Failure Modes to Prevent

### Failure Mode 1: Creative before commercial logic
Fix:
- route upward to offer or tactical layer first

### Failure Mode 2: Too many channels too early
Fix:
- reduce to one primary and one supporting channel

### Failure Mode 3: Proposal work without offer clarity
Fix:
- lock offer and buyer before narrative work

### Failure Mode 4: Content volume without funnel role
Fix:
- assign each asset a stage and CTA

### Failure Mode 5: Project coordination without single truth
Fix:
- query Context Keeper and restart handoffs from aligned context

## The Operating Standard

The system is considered correctly launched when:
- every major workflow begins with orchestration and context
- every specialist receives a real brief
- every asset has a job and destination
- every output feeds the next handoff
- every cycle creates usable learning

That is the difference between a business system and a very enthusiastic content machine.
